Virtualenv is used to manage Python packages for different projects. Using virtualenv allows you to avoid installing Python packages globally which could break system tools or other projects.

1. Open Command Prompt
    Process : Click on windows search bar --> Search 'cmd' --> Click on 'Command Promt App'
2. Move to Desktop location
    Command : cd Desktop Press enter

3. Install Virtualenv
    Command : pip install virtualenv Press enter
4. If you already installed virtualenv, it will show like this

1. Create one folder on desktop with the name 'myBlog'.
    Process : Right click on Desktop --> Click on 'New' --> Click on Folder --> type the name as 'myBlog'
2. Open that folder in VsCode.
    Process : Right click on that folder --> Click on 'open with vscode'.
3. Once VsCode opens, open new terminal
    Process : click on Terminal --> New Terminal
4. Create new command prompt terinal and delete powershell terminal.
    Process : Click on '+' symbol --> click on Command Prompt --> click on delete button of powershell terminal
5. Create new virtual environment for this project. (Note* : We need to create one new virtual environment for each new project.)
    Command : virtualenv blog Press enter
6. Activate virtual environment (Note* : We need to activate this venv, whenever we start working on this project.)
    Command : blog\Scripts\activate Press enter

1. Install django using this below command.
    Command : pip install django Press enter
2. Execute this below command to collect all the packages that we installed for this project. So this will help us whenever we share this project to others to work in a team. They can easily install exact version of the packages we installed for this project.
    Command : pip freeze > requirements.txt Press enter
    More* : If you share your project with others, they just need to do two steps.
    1. Download your project, go to project folder, open command prompt.
    2. Run this command pip install -r requirements.txt Press enter

1. Run the command django-admin startproject projectName to create new django project. Remember 'projectName' can only contain letters, numbers and underscore.
    Command : django-admin startproject personal_blog Press enter
In the above picture, you can see new folder has been created with the project name that you have entered. In that folder, another folder has been created with the same name. That folder, in django we call it as 'app'. It includes 5 files, that we will see in the next section.
2. Now run your first django project.
    For that you need to move inside project folder.
    Command : cd personal_blog Press enter
    Now execute the project run command.
    Command : python manage.py runserver Press enter
